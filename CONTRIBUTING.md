# Contributing to go-go-host

Start with [`docs/contributing/README.md`](docs/contributing/README.md).

That guide explains the system, helps you choose the right contribution lane, and points to the backend, runtime, dashboard, testing, local-development, and docmgr workflows. The short version is: learn the layer you are changing before you edit it, keep product invariants in the control services, validate with the lane-specific checklist, and leave documentation behind when you discover a new pattern.
